# CLAUDE.md

Guidance for Claude Code (and any contributor) working in this repo. Tracked in git so it's visible to PR authors and agent sessions on any machine.

## What this is

A public **knowledge commons / wiki** for **Attention Commons** — attention as a shared resource being enclosed by the attention economy, inside the broader frame of **Culture of Attention (Культура внимания)**. Heavy on contemplative traditions (meditation/lineages) and integral theory (Wilber). Joint project of Artem Zhiganov and Viktor Shiryaev (student of Shinzen Young; well-known meditation teacher in Russia/Ukraine).

- Live: https://attention-commons.netlify.app (canonical https://attentioncommons.org once DNS is connected — see issue #1).
- Sibling repo for the working space (not for publication): https://github.com/zhiganov/active-solidarity.

## Stack & build

- **Astro 6 + Starlight** (`@astrojs/starlight`), static output.
- Content: Markdown/MDX in `src/content/docs/`, organised into the sidebar groups in `astro.config.mjs`.
- Self-hosted fonts via `@fontsource-variable/source-serif-4` + `@fontsource/ibm-plex-mono` (no external font requests).
- Build: `npm run build` → `dist/`. Pagefind builds the search index automatically.
- Deploy: **Netlify**, config in `netlify.toml`. `NODE_VERSION = "22"` is required — **Astro 6 needs Node ≥22.12**; the Netlify default (20) breaks the build (Astro aborts with `Node.js v20.x is not supported`). If you ever bump Astro/dependencies, re-check the engines and bump `NODE_VERSION` to match.
- Local dev: `npm run dev` → http://localhost:4321.

## Hard guardrails (read before editing content)

1. **Viktor's authored writing is not published without his explicit sign-off.** The *Culture of Attention* manifesto exists in draft as material for his book; the wiki currently represents it only by its *moves* on `idea/commons-vs-ground.md` with a `:::caution[Manifesto pending]` block. Do **not** replace the placeholder with the manifesto text until issue #4 is closed by Viktor's go-ahead.
2. **Nothing private.** No content from `active-solidarity/docs/private-notes.md` (people-pipeline, health, money, private relationships) reaches this site. Public, published references and public fellow-travellers only.
3. **No AI-generated brand assets.** Logos, OG/illustration art, favicons → designer, stock, or AI-gen on the project side; not generated by an agent in-session. See issue #3.
4. **No AI attribution** in user-facing copy (commit trailers are fine).
5. **Voice:** sober, scholarly, provisional ("not a finished position"). Treat contemplative material with depth, not generic wellness framing. See `PRODUCT.md`.

## Design system

- `PRODUCT.md` (register, users, principles, anti-references) and `DESIGN.md` (literary press-editorial: warm paper + ink, evergreen accent, Source Serif 4) at the repo root. Both are the Impeccable context files.
- **Impeccable is the design tool for this repo.** For any visible UI change, use Impeccable sub-commands (`/impeccable critique`, `/polish`, `/layout`, `/colorize`, `/typeset`, `/adapt`) — not inline CSS tweaks, and not generic `frontend-design`. See `feedback_use_impeccable_subcommands` in `claude-config/memory/`.
- Theme tokens live in `src/styles/custom.css` (Starlight CSS custom-property overrides). Asides are de-striped per Impeccable's absolute-ban list.

## Content writing

- One topic per Markdown file (Starlight convention).
- Cite sources — every substantive claim should be traceable to a named, public reference. When uncertain, leave it general or out.
- Source material for the existing flagship pages comes from `../active-solidarity/{research,concept,docs/development-log}.md`. Read those before extending those pages, but **do not** copy in private/log material wholesale.
- 13 of the 18 pages are intentional stubs marked with `:::note[Draft]` tied to tracking issues. Fill them as Viktor's source materials arrive (see issue #5).

## Don't

- Don't commit `node_modules/`, `dist/`, or anything in `.env*`.
- Don't add Astro components or layout overrides for theming — overrides go in `src/styles/custom.css` only.
- Don't open a custom `404.md` under `src/content/docs/` — it conflicts with Starlight's built-in `/404` route and is overridden. Starlight's themed 404 is already wired.
- Don't change the `editLink.baseUrl` away from `main` without also renaming the GitHub default branch.
